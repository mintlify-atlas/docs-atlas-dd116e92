## Overview

The Dispatch API provides endpoints for managing 911 calls, BOLOs (Be On the Lookout), active dispatchers, area of play, and unit status updates.

<Info>
Most dispatch endpoints require the `Dispatch` permission.
</Info>

---

## Get Dispatch Data

<CodeGroup>
```bash GET /api/dispatch
curl -X GET https://your-cad.com/api/dispatch \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Get current dispatch information including area of play and active dispatcher count.

### Response

<ResponseField name="areaOfPlay" type="string">
  Current area of play (AOP)
</ResponseField>

<ResponseField name="activeDispatchersCount" type="number">
  Number of active dispatchers
</ResponseField>

<ResponseField name="userActiveDispatcher" type="object">
  Current user's active dispatcher status
</ResponseField>

## Set Dispatcher Active/Inactive

<CodeGroup>

### Response

<ResponseField name="dispatcher" type="object">

<ResponseField name="activeDispatchersCount" type="number">

## 911 Calls

### Get All 911 Calls

<CodeGroup>
```bash GET /api/911-calls
curl -X GET "https://your-cad.com/api/911-calls?includeEnded=false&skip=0" \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Get all 911 calls with filtering options.

#### Query Parameters

<ParamField query="includeEnded" type="boolean" default="false">
  Include ended calls
</ParamField>

<ParamField query="skip" type="number" default="0">
  Pagination offset
</ParamField>

<ParamField query="query" type="string">
  Search query
</ParamField>

<ParamField query="department" type="string">
  Filter by department ID
</ParamField>

<ParamField query="division" type="string">
  Filter by division ID
</ParamField>

<ParamField query="assignedUnit" type="string">
  Filter by assigned unit ID
</ParamField>

#### Response

<ResponseField name="totalCount" type="number">
  Total number of calls
</ResponseField>

<ResponseField name="calls" type="array">
  Array of 911 call objects
</ResponseField>

### Get Call by ID

<CodeGroup>
```bash GET /api/911-calls/:id
curl -X GET https://your-cad.com/api/911-calls/{callId} \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Get a specific 911 call by ID or case number.

#### Path Parameters

<ParamField path="id" type="string" required>
  Call ID or case number
</ParamField>

### Create 911 Call

<CodeGroup>
```bash POST /api/911-calls
curl -X POST https://your-cad.com/api/911-calls \
  -H "Cookie: snaily-cad-access-token=<token>" \
  -H "Content-Type: application/json" \
  -H "is-from-dispatch: true" \
  -d '{
    "name": "John Doe",
    "location": "123 Main Street",
    "postal": "90210",
    "description": "Suspicious activity reported",
    "type": "call-type-id",
    "situationCode": "situation-code-id"
  }'
```
</CodeGroup>

Create a new 911 call.

#### Headers

<ParamField header="is-from-dispatch" type="string">
  Set to "true" if call is created by dispatch (bypasses approval)
</ParamField>

#### Request Body

<ParamField body="name" type="string">
  Caller name
</ParamField>

<ParamField body="location" type="string">
  Incident location
</ParamField>

<ParamField body="postal" type="string">
  Postal/zip code
</ParamField>

<ParamField body="description" type="string">
  Call description
</ParamField>

<ParamField body="type" type="string" required>
  Call type ID
</ParamField>

<ParamField body="situationCode" type="string">
  Situation code ID
</ParamField>

### Update 911 Call

<CodeGroup>
```bash PUT /api/911-calls/:id
curl -X PUT https://your-cad.com/api/911-calls/{callId} \
  -H "Cookie: snaily-cad-access-token=<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Updated call details",
    "postal": "90211"
  }'
```
</CodeGroup>

Update an existing 911 call.

### End 911 Call

<CodeGroup>
```bash PUT /api/911-calls/:id/end
curl -X PUT https://your-cad.com/api/911-calls/{callId}/end \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Mark a 911 call as ended.

## BOLOs

### Get All BOLOs

<CodeGroup>
```bash GET /api/bolos
curl -X GET "https://your-cad.com/api/bolos?skip=0&type=VEHICLE" \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Get all Be On the Lookout (BOLO) entries.

#### Query Parameters

<ParamField query="skip" type="number" default="0">
  Pagination offset
</ParamField>

<ParamField query="includeAll" type="boolean" default="false">
  Include all BOLOs
</ParamField>

<ParamField query="query" type="string">
  Search query
</ParamField>

<ParamField query="type" type="string">
  BOLO type: `VEHICLE`, `PERSON`, or `OTHER`
</ParamField>

#### Response

<ResponseField name="bolos" type="array">
  Array of BOLO objects
</ResponseField>

<ResponseField name="totalCount" type="number">
  Total number of BOLOs
</ResponseField>

### Create BOLO

<CodeGroup>
```bash POST /api/bolos
curl -X POST https://your-cad.com/api/bolos \
  -H "Cookie: snaily-cad-access-token=<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "VEHICLE",
    "description": "Stolen vehicle",
    "plate": "ABC123",
    "model": "Honda Civic",
    "color": "Red"
  }'
```
</CodeGroup>

Create a new BOLO.

#### Request Body

<ParamField body="type" type="string" required>
  BOLO type: `VEHICLE`, `PERSON`, or `OTHER`
</ParamField>

<ParamField body="description" type="string" required>
  BOLO description
</ParamField>

<ParamField body="plate" type="string">
  License plate (for vehicles)
</ParamField>

<ParamField body="model" type="string">
  Vehicle model
</ParamField>

<ParamField body="color" type="string">
  Vehicle or person color/description
</ParamField>

<ParamField body="name" type="string">
  Person name (for person BOLOs)
</ParamField>

### Update BOLO

<CodeGroup>
```bash PUT /api/bolos/:id
curl -X PUT https://your-cad.com/api/bolos/{boloId} \
  -H "Cookie: snaily-cad-access-token=<token>" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Updated BOLO information"
  }'
```
</CodeGroup>

Update an existing BOLO.

### Delete BOLO

<CodeGroup>
```bash DELETE /api/bolos/:id
curl -X DELETE https://your-cad.com/api/bolos/{boloId} \
  -H "Cookie: snaily-cad-access-token=<token>"
```
</CodeGroup>

Delete a BOLO.
